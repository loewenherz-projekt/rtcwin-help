The Programming Language RTC     Functions and Procedures

Principal Structure

    

Functions are program parts realizing certain repetitive tasks. Once declared, the functions are later called up by their names. Thus, they significantly contribute to a good structure and readability of a program.

The principal design of a function is as follows:

Type Name (Parameters)

{

  Declarations

  Statements

  return (return value);

}

 

Type determines the data type of the function or – to be precise – the data type of the return value. Following the Name of the function a list of parameters is added in a pair of parentheses. After that, the function code is appended in a pair of curved brackets. Typically, the locally needed variables are declared first, followed by the statements making up the function. The statement return terminates the function.

·The indication of parameters is made – like the declaration of variables – via data type and parameter name.
·Multiple parameters in a list are separated by a comma.
·If no parameters are needed, the word void is written instead.
·Declaration of local variables is optional and may be omitted.
·The return value is optional and has to comply with the data type of the function. If the return value is missing, also no function result is passed to the calling routine (void).
·The statement return is optional and may be omitted for statements without return value. In that case, the function is terminated after execution of the last statement.
·The statement return can be used multiple times within a function (e.g. in an if-else struct). The function is terminated immediately after the execution of return – subsequent instructions are discounted.
·Recursive function calls (function calling itself) are not permitted.

RTC does not distinguish between functions and procedures. Instead of a procedure, a function without return value is defined by stating void instead of a data type.
