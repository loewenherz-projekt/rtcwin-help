fopen

    

RTC Function

Opens a file.

Declaration:

int fopen (int fno, string fname)

Group:

File handling

Parameters:

fno
fname

Logical file number
File name

Return value:

Error code

 

The function fopen opens a file and makes it available under the provided logical file number fno) for subsequent read and write operations (e.g. fwrite). The logical file number fno can freely be chosen from the range 0 to 7.

The parameter fname contains the full file name as follows:

internal/name.ext

The file name with the extension ext is contained in the battery-backed memory of the weighing terminal and cannot be accessed externally.

shared/name.ext

The file name with the extension ext is contained in the directory "shared". This folder can be accessed externally via FTP access (login: "shared," password as set in Service Mode).

usbstick/.../name.ext

The file name with the extension ext is contained on a USB memory stick. Further subdirectories may be specified as applicable.

Files on the USB memory stick can be accessed externally via FTP (login: "usbshared," password as set in Service Mode).

sharename/.../name.ext

The file name with the extension ext is contained on a network drive. sharename is the symbolic path name as it was returned by the instruction smbconnect. Further subdirectories may be specified as applicable.

If the file name does not specify a directory, the folder 'internal' is used as default.

If the specified file does not exist, it is created by fopen. If the file can be opened successfully, fopen returns 0 as function result; otherwise the corresponding error code is returned instead.

 

Example:

/* Write weighing data to log file */

bool WriteWeighingDataToLogFile (string directory, string data)

{

int result = 0;

  if (fopen (1, concat (directory, "WeighLog.csv")) == 0)    // directory = "shared/"

  {

    result = fwrite (1, data, length (data));                // data = "date;gross;net"

    if (fclose (1) > 0)

    {

      ShowErrorMsg ("Error log file!");

      return (false);

    }    

 

    if (result == 0)                                         // Evaluate variable result

    {

      ShowErrorMsg ("Error log file!");

      return (false);

    }

    else if (result != length (data))

    {

      ShowErrorMsg ("Storage full1!");

      return (false);

    } 

    else

      return (true);

  }

  else  

  {

    ShowErrorMsg ("Error log file!");

    return (false);

  }

}

 

 

Notes:

After opening a file, the internal read/write pointer is set to the end of the file.

Depending on type and performance of the connection, opening of a file may take several seconds. The programmer should respect this in his program design.
