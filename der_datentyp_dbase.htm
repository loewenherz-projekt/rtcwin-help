The Data Type dbase

The indications "12015", "Apples, Boskoop", "1.15" and "24.72" are interrelated and form a so-called record. This record is stored in a table with corresponding structure as shown above.

In RTC, database tables are constituted by their own data type. The corresponding declaration word is dbase, followed by a name as applies to all (variable) declarations.

dbase "Article"   Article[100]

{

   "Article No."  decimal  No%3.0;

   "Name"         string   Name[25];
   "Price/kg"     decimal  Price%6.2;

   "Consumption"  decimal  Total%8.dp;

}

The name of the database table may precede an optional display text used for the database function edit. If this text is missing, the table name is shown instead for the execution of edit. The name is followed – in square brackets – by the maximum number of table entries (records) and – in curved brackets – by the declaration of the individual table fields.

The declaration of the table fields is identical to the declaration of variables. Again, an alternative display text may be specified for the execution of edit and again, the field name is used as a replacement where the optional display text is missing.

The table field declared in first place automatically serves as index field. With other words, the records in the table above are stored against their article number. There are no two records with identical article number.
