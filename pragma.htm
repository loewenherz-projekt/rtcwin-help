#pragma

    

RTC Function

Preprocessor control statement.

Declaration:

#pragma ( key, value )

Group:

Preprocessor

Parameters:

key
value

Name of keyword
Assigned value

 

With the #pragma statement, certain preprocessor control statements can be embedded in the application program. The #pragma statement is followed by a pair of brackets containing the name of a key and – separated by a comma – the value of the key. At the moment only the key 'language' is defined which is assigned a language string.

The language string contains – separated by semicolons – the setup for several languages. Each setup consists of a text, the number of the required ISO 8859 character set and the country code for the layout of an external PC keyboard. If applicable, multiple keyboard layouts can be stated using square brackets.

The preprocessor uses the given language string on the one hand to initiate the compiler for using the right ISO code pages and on the other hand to embed a writeparameter statement in the function main that configures the language selection of the Service Mode accordingly.

 

Example:

//  available languages:    German|English

#pragma (languages, "German,15,[de,ch];English,15,[us,gb];")

 

#define TX_ABORT               "Abbruch|Abort"

#define TX_ABORT_BATCH         "Charge abbrechen?|Abort batch?"

 

Notes:

If the translate statement is used, the name of the language in the #pragma statement has to be identical to the identifier of the corresponding language in the translate.ini file.
