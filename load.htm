load

    

RTC Function

Loads a database table from a csv file.

Declaration:

bool load (dbase db, string fname)

Group:

Database functions

Parameters:

db
fname

Database table
File name

Return value:

Function result

 

The function load fills the database table specified by the parameter db with the content of a CSV file. Any records already contained in the database table are deleted. The file has to contain the records concluded by carriage-return and line-feed characters with the individual table fields separated by semicolons. The parameter fname contains the full file name as follows:

internal/name.ext

The file name with the extension ext is contained in the battery-backed memory of the weighing terminal and cannot be accessed externally.

shared/name.ext

The file name with the extension ext is contained in the directory "shared". This folder can be accessed externally via FTP access (login: "shared," password as set in Service Mode).

usbstick/.../name.ext

The file name with the extension ext is contained on a USB memory stick. Further subdirectories may be specified as applicable.

Files on the USB memory stick can be accessed externally via FTP (login: "usbshared," password as set in Service Mode).

sharename/.../name.ext

The file name with the extension ext is contained on a network drive. sharename is the symbolic path name as it was returned by the instruction smbconnect. Further subdirectories may be specified as applicable.

If the file name does not specify a directory, the folder 'internal' is used as default.

 

Example:

/* Load articles from USB memory */

bool result = load (Products, "usbstick/Products.csv");

 

Notes:

The function load includes file open and close. Calling fopen or fclose is not necessary.

The function load is a method of the declaration word dbase.
