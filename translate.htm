translate

    

RTC Function

Shows a string in a specific language.

Declaration:

string translate (string s, [string language])

Group:

String functions

Parameters:

s
language  

String
Optional language

Return value:

Result string

 

The function translate returns as result a language-dependent string according to the settings specified by the parameter language.The result string is copied from the file 'translation.ini' contained in the folder shared or appdata of the weighing terminal. The structure of the file is as follows:

[English]
TX_CANCEL  "Cancel"
TX_START     "Start"
 
[German]
TX_CANCEL  "Abbruch"
TX_START     "Start"

Language identifier in square brackets
Index (TX_CANCEL) and corresponding text
More texts in the same language
 
Next language identifier
Texts in another language

The parameter s serves as index when searching the file translation.ini during the execution of translate. The function only searches in the group specified by the parameter language. If the parameter language is missing, the language set in Service Mode is searched instead. If the requested index cannot be found in the indicated language, it is searched in the language stated first in the pragma statement (e.g. English).

 

Example:

showtext (20, 10, TX_ARTICLE_FILE);              // Display language as set in Service Mode

 

showtext (20, 10, translate (TX_CONFIGURATION, "English"));  // Display in English, independent of language settings

 

 

Notes:

The function translate interprets the name of the language set in Service Mode. The pragma statement determines the selectable language options in the Service Mode.

In a program, the function lsub can be substituted with translate.
