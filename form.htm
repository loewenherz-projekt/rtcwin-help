...
}

The instruction form may be used for the declaration of a print format. The declaration word is followed by the name of the print format and – embraced in curved brackets – the individual print fields. Each print field consists of a position specified by line number lno and column number cno and the literal print field field. field can be a fix text or a variable.

The print field may be assigned one or more of the following attributes. Several attributes have to be separated by commas.

     bold

Emphasized printing in bold characters

     underlined

Underlined characters

     condensed

Printing with reduced width

     italic

Printing in italics

If the required print attribute is not available (e.g. inverse, barcode, etc.), the corresponding printer command sequence can be entered directly. The control sequence seq0 precedes the print field and contains the command to activate the attribute. Correspondingly, the sequence seq1 follows the print field and is used to deactivate the attribute. The individual characters of the control command are entered in decimals, separated by commas and enclosed by angle brackets, e.g. "<27,65,66>" for the sequence "Esc A B".

 

Example:

form WeightTemplate

{

    1   1 date 

    1  10 time 

    1  16 "Scale" 

    1  22 ScaleNo

    2   1 "Gross"   

    2  10 fgross bold 

    3   1 "Tare"    

    3  10 ftare  bold

    4   1 "Net"

    4  10 <27,87,49> fnet <27,87,48>  /* Expand: ESC W 1/0 */

    8   1 " " 

}

 

Note:

The print formats created by form can later be edited via PC without access to the program code, e.g. using IT CONFIGURATOR.
