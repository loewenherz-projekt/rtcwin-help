The Programming Language RTC     Printout Formatting

Embedding of External Layouts

    

To produce a printout with the inclusion of an externally created layout, simply pass the instruction print the name of the layout file as third parameter:

    showtext (20, 20, "Printing...          ");

    print (Port, Ticket, "Sample.lab");                     // Output label on channel prt

The stated layout file contains the layout of the ticket as it was created with an applicable label-creating program (e.g. NiceLabel®) to match with the selected printer. Where the layout file contains any text enclosed in tildes '~' (e.g. '~date,10~'), this text is replaced by the content of the variable with identical name, when the layout is sent to the printer. The value behind the comma specifies the length of the printer output. Thus, a reference to the variables of the application program can be made when the layout is created.

The instruction print does not need the file name as a third parameter if the layout file and the print format have the same name (plus file extension '.lab').

If the instruction print is missing, the name of the layout file and/or if the layout file cannot be found, the stated print format serves as a default.

The layout file can only access the variables of the corresponding print format.
