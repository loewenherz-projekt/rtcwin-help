fseek

    

RTC Function

Moves the internal read/write position in an opened file

Declaration:

int fseek (int fno, int pos, int mode)

Group:

File handling

Parameters:

fno
pos
mode

Logical file number
Relative position the read/write pointer is moved to.
Initial point for moving the pointer:
0 = Beginning of file
1 = Current position
2 = End of file

Return value:

Error code

 

The instruction fseek moves the current read/write position of the file specified by the logical file number fno to the relative position specified by the parameter pos. The parameter mode specifies the initial point the relative position refers to as follows: 0 = beginning of file; 1 = current position; 2 = end of file. The file has to be opened first using the instruction fopen. If the read/write position can be moved successfully, fseek returns 0 as function result; otherwise the result is -1.

 

Example:

/* Read parameters from USB memory */

bool ReadFile (string directory)

{

string data_buffer[1023] = ""; 

 

  if (fopen (1, concat (directory, "WeighLog.csv")) == 0) // directory = "usbstick/"

  {

    if (ftell (1) > 0)                                    // Read current read/write position

    {

      if (fseek (1, 0, 0) == 0)                           // Move cursor to the beginning of file 

      {

        while (fread (1, data_buffer, 1023) > 0)          // Read data from file

        {

          /* Process parameter  */

        }

        return (true);

      }  

      else

      {

        ShowErrorMsg ("Zugriff verweigert");

        return (false);

      }

    } 

    else

    {

      ShowErrorMsg ("Zugriff verweigert");

      return (false);

    }   

  }

  else  

  {

    ShowErrorMsg ("Fehler Log-Datei");

    return (false);

  }

}

 

Note:

After opening a file, the internal read/write pointer is always set to the end of the file.
