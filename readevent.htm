readevent

    

RTC Function

Reads a pending event.

Declaration:

bool readevent (int *source, int *type, int *id, string *data[])

Group:

Touch screen programming

Parameters:

source
type
id
data

Variable containing the event source
Variable containing the event type
Variable containing the id of the item that raised the event
String array for the event arguments

Return value:

result of type bool

 

The function readevent reads the next pending event from the event buffer. If a not yet handled event is available, readevent writes the source, type and identification number of the object into the corresponding variables without delay. Any potential arguments are written into the string array data after the execution of readevent. If there is no event pending, program execution waits until a new event is raised. readevent returns true as function result.

The function eventreceived can be used to check the availability of an event prior to the execution of readevent.

 

Example:

void WeighingApplication (void)

{

int source, type, ident;

string event_data[4][255];

 

  selectpage (PAGE_WEIGHING_MODE);

  dropevents (); // Drops all unprocessed events.

  for (;;)

  {

    if (eventreceived() > 0) // Get all events...

    {

      readevent (source, type, ident, event_data); 

      if (source == EVENT_CLASS_APPLICATION) // Only events raised by the application...

      {

        switch (type)

        {

          case EVENT_TYPE_BUTTON_PRESS : 

                     switch (ident)

                     {

                       case ID_BUTTON_0 : return; // Pressed Button_0 return to main menu

                                          break;

                       case ID_BUTTON_1 : /* ... */

                                          break;

                     }

                     break;

          

          case EVENT_TYPE_TEXT_ENTERED :

                     /* ... */

                     break;

          /* ... */

        }

      }

    }

    /* ... */

    delay (0.1);

  }

}

 

Note:

Events may be raised from different sources and it cannot be assumed that a pending event was raised by the application. This has to be considered in the design of the program.
