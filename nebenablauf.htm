The Programming Language RTC     Multitasking

Background Task

    

To assign further functionality to the one fix background task, simply create a function without parameters and name it 'background_task.'

void background_task (void)

{

 

}

 

The fact alone that the function is given this reserved name assures that it is linked to and executed by the fix system background task.

Into the function background_task, you put all the functions that you need to be executed in parallel to the application program. In doing so, the following needs to be taken into account:

The function background_task must not contain any loops. In particular, the background_task itself must not be an endless loop as it would not terminate anymore once it is called up and thus block the complete system.

The function background_task must not contain any delays or call functions containing possible delays, as this would delay execution of the complete background task.
