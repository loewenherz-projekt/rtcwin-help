The Programming Language RTC     XML Interface

XML Syntax

    

Basically, an XML instance or file consists of the root element which then contains all other elements. Those elements may again include more elements or plain text. Elements that contain other elements are called nodes.

The following rules apply to the names of nodes:

·XML is case sensitive, i.e. a difference is made between upper and lower case characters.
·All characters may be used, including special characters and umlauts.
·The node must be closed.

 

Example:

<root>
  <Text_1>
    Here some text
  </Text1>
  <Text_2>
    Some more text
  </Text_2>
</root>

Root element or root node
First sub-element
 
End of first sub-element
Second sub-element
 
End of second sub-element
End of root element

The above example can programmed using xmlinsertchild:

#define HANDLE   0;
 
xmlclear (HANDLE);
xmlinsertchild (HANDLE, "", "root", 1, 1);                      // root element
xmlinsertchild (HANDLE, "root", "Text_1", 1, 1);                // 1st sub-element
xmlinsertchild (HANDLE, "root/Text_1", "Here some text", 2, 1); // 1st text
xmlinsertchild (HANDLE, "root", "Text_2", 1, 1);                // 2nd sub-element
xmlinsertchild (HANDLE, "root/Text_2", "Some more text", 2, 1); // 2nd text

 

The above example can be written even more easily using the instruction xmlsetnodevalue utilizing the fact that not yet existing nodes are created automatically:

#define HANDLE   0;
 
xmlclear (HANDLE);
xmlsetnodevalue (HANDLE, "root/Text_1", "Here some text"); // 1st text
xmlsetnodevalue (HANDLE, "root/Text_2", "Some more text"); // 2nd text

 

This also clearly shows the path structure. When the second sub-element is added, the first node is automatically closed because the name "Text_1" is not contained in the path of the second node. If "Text_1" were contained in the path of node Text_2, it would be inserted as sub-element of node Text_1. Example:

// ...
xmlinsertchild (HANDLE, "root/Text_1", "Text_2", 1, 1); 
// ...

 

or:

// ...
xmlsetnodevalue (HANDLE, "root/Text_1/Text_2", "Some more text"); 
// ...

 

Thus, the example would look like this:

<root>
  <Text_1>
    Here some text
    <Text_2>
      Some more text
    </Text_2>
  </Text1>
</root>

Root element or root node
First sub-element
 
Second sub-element
 
End of second sub-element
End of first sub-element
End of root element

The Programmer does not need to explicitly close a node. Nodes are closed automatically when the path is cut accordingly. The root element is closed when the XML instance is saved or transmitted.
