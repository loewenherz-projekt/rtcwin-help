The instruction dbase declares a database table with given name. The name is followed – in square brackets – by the maximum number of table entries (records) and – in curved brackets – by the declaration of the individual table fields. The table name may precede an optional display text for the function edit. If it is missing, the table name is displayed instead.

The declaration of a table field complies with the declaration of a variable which is also given the optional operator prompting for displaying and editing the field. For table fields all types bool, char, int, float, decimal and string are allowed. Again, the operator prompt can be omitted; a combination of table and field names is displayed in that case. Each database field can be addressed with its unique identifier consisting of the table name, a full stop and the field name, e.g. table.fieldname.

The table field declared first automatically serves as index field, i.e. all entries are stored against this first field. When a record is inserted, an existing record with identical index field is overwritten. The table field declared in second position serves as additional sort criteria for the function select and as an optional search field for the operation inputbox.

If applicable, the individual table fields may be assigned the control commands selectmode and editmode. The instruction selectmode controls the database function select as follows:

 selectmode = input

During execution of select, the record can be edited/created after pressing the F4 key. If a value other than zero is entered in the index field, the newly created record is inserted in the database table. If zero is entered, the entered data is not stored and only temporarily available.

The instruction editmode controls the database function edit as follows:

 editmode = showonly

The table field is only shown and cannot be edited.

 editmode = skip

The table field is invisible during input.

 editmode = select(db_name)

The entry for this database field must be a selection from another database db_name. Free data entry is not allowed.

Furthermore, editing of a database field via the function edit can be controlled through the following instructions:

 range = expr

The table field input is checked. The entry is only accepted if the expression expr returns true.

 skipcondition = expr

Entry for the table field is only enabled if the expression expr returns false. If the expression returns true, the table field is disabled and entry is skipped.

Notes:

The above listed instructions may be assigned to multiple table fields. The instruction "selectmode = input," however, is only applicable for the first field.

Fields with "editmode=skip" have to be put to the end of the table.

Example:

dbase "Customer"    Clients [100]

{

   "Customer No."   decimal  No%3.0;     selectmode = input;

   "Name"         string   Name[25];
   " "            string   Name2[25];  skipcondition = Clients.Name == "";
   "Street"      string   Street[25];

   "ZIP Code"          string   Zip[5];     range = Clients.Zip != "";

   "City"          string   City[25];

   "Total"        decimal  Total%8.dp; editmode = skip;

}

 

Notes:

The declaration of a database table supports multi-lingual prompting. The use of the function lsub is not necessary.

All table fields are task local variables; the database table itself is stored in the non-volatile storage of the weighing terminal.
