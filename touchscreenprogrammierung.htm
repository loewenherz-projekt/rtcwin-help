The Programming Language RTC     Touch Screen Programming

General

    

Applications for weighing terminals with a touch screen are fundamentally different from applications for classic indicators.

The classic application is sequentially structured and executed step-by-step in a fix order. For example, the operator first chooses the weighing mode and then enters a batch number and selects an article. Next, he is prompted to put an empty container onto the scale. Then the scale is tared and so on. The program flow is clearly defined by the program structure and cannot be changed.

On a touch-screen indicator the operator can choose from a selection of objects on the screen (the widgets) on which he can tap with his fingers to activate them. Whether first the batch number is entered or the article is selected is up to the operator and not predetermined by the program structure. The application is "event-driven."
