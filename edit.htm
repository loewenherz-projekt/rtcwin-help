edit

    

RTC Function

Input dialog for database tables.

Declaration:

void edit (dbase db)

Group:

Display and keyboard

Parameters:

db

Name of database table

Return value:

-

 

The instruction edit generates a complete dialog for creating/editing records of a database table. The structure of the dialog – as it appears on the application screen – complies with inputbox. The sequence of field variables and operator prompts complies with the declaration of the database table using dbase. As a feedback for the operator, the title line also contains the capacity of the database table and the number of records contained herein.

The function edit defines five function keys as follows:

F1

Enable editing for the input field (only for data type string)

F2

Dump the database table on a line printer on COM1 (see note)

F3

Select a record from a listing, sorted by the search field (see also select)

F4

Delete the current record from the table (after confirmation)

F5

Display an optional help text (see setinfo)

F6

Exit dialog

 

Example:

dbase "Article file"  Article[99]

  {

    "Article No."   decimal  No%2.0;

    "Name"          string   Name[24];

    "Consumption"   decimal  Total%8.dp;

  }

 

void SupervisorMode (void)

{

  if (codebox ("Password", UserPassword) == ok)

    selectbox ("Settings")

    {

      "General"      GeneralSettings();

      "Articles"     edit (Article);

    }

}

 

 

Notes:

The instruction edit provides multi-language operator prompting; the use of the function lsub is not necessary.

The instruction edit does not save the original content of the application screen or the previously selected font attributes. This has to be considered in the design of the program.

The default print channel may be changed using the instruction writeparameter.
