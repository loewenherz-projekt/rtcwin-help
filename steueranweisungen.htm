The Programming Language RTC     Preprocessor

Control Statements

    

Through the directive

#pragma (key, value)

certain preprocessor control statements can be embedded in the application program. The #pragma statement is followed by a pair of brackets containing the name of a key and – separated by a comma – the value of the key. Currently, only the following key is defined:

Key

Value

Description

languages

Language string, e.g.
"Deutsch,15,de;English,15,[gb,us]"

Installation of language, character set and keyboard layout.

The language string contains – separated by semicolons – the setup for several languages. Each setup again consists of a text, the number of the required ISO 8859 character set and the layout of the external PC keyboard. If applicable, multiple keyboard layouts can be stated using square brackets.

The preprocessor uses the given language string on the one hand to initiate the compiler for using the right character set for the given texts and on the other hand to embed a writeparameter statement in the main function that configures the language selection of the Service Mode accordingly.
