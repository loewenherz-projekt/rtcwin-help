Handling Events

    

Most of the objects (widgets) can raise events when activated by the operator. The type of the event generally depends on the kind of object. As soon as a button is activated, it generates an event of type EVENT_TYPE_BUTTON_PRESS, whereas a text input field raises an event of type EVENT_TYPE_TEXT_ENTERED as soon as data entry is finished.

Following is a list of events that can be raised by the application (EVENT_CLASS_APPLICATION):

Event

Description

EVENT_TYPE_BUTTON_PRESS

A button was pressed.

EVENT_TYPE_TEXT_ENTERED

The input into an entry field was completed. Contains the entered text as argument.

EVENT_TYPE_LIST_ITEM_SELECTED

An item from a list was selected. Contains the ID and text of the list element as arguments.

EVENT_TYPE_TABLE_ITEM_CHANGED

A field of a table was modified. Contains column and row number of the field and the new text as arguments.

EVENT_TYPE_CHECKBOX_CHANGED

A checkbox was activated. Contains as argument if the box is checked (true) or unchecked (false).

EVENT_TYPE_COMBOBOX_ACTIVATED

An item was selected from a drop-down menu. Contains the number and text of the selected item as argument.

EVENT_TYPE_SKETCH_PRESSED

The surface of an image (outputbox) was activated. Contains as arguments the x and y coordinates of the point that was touched.

EVENT_TYPE_MESSAGEBOX_BUTTON

One of the function keys in the messagebox was pressed. Contains as argument the number of the button that was pressed.

EVENT_TYPE_SKETCH_SIZE_CHANGED

The size of an image (WIDGET_TYPE_SKETCH) was changed. Contains the new width and height as arguments, so the application can re-arrange the items. The SIZE_POLICY of this object may not be set to 'fixed' in this case.

EVENT_TYPE_TEXT_ABORTED

The text entry was aborted.

EVENT_TYPE_TEXT_DROPDOWN_ENTERED

A text was entered into the drop-down menu of the keypad. Contains as argument the text that was entered.

EVENT_TYPE_TEXT_DROPDOWN_CHANGED

The text in the drop-down menu was changed (e.g. a character was entered or deleted). The modified text is returned as argument.

EVENT_TYPE_TEXT_DROPDOWN_ABORTED

The entry into the drop-down menu was aborted.

EVENT_TYPE_USER_TEXT_INPUT_CLICKED

An object of the type WIDGET_TYPE_TEXT_INPUT_USER was activated. The application could react on this e.g. by opening the onscreen keypad.

Furthermore, events can also be generated by the monitor (EVENT_CLASS_MONITOR). In detail:

Event

Description

EVENT_TYPE_STATUS_BUTTON

A field in the status bar at the top of the screen was activated (e.g. time or battery status). Contains as argument the number of the field clicked.

EVENT_TYPE_CONNECTION_ESTABLISHED

A new monitor has connected to the application. The application may need to redraw or create the screen content.

EVENT_TYPE_INDICATED_SCALE_CHANGED

The indicated scale was changed. Contains the number of the now indicated scale as argument. See also indicatedscale.

 

Example:

void main (void)

{

int source, type, ident;

string data[4][1023];

 

  selectpage (PAGE_1);

  for (;;)

  {

    // wait for next event:

    while (eventreceived())

    {

      readevent (source, type, ident, data);

      if (Source == EVENT_CLASS_APPLICATION) 

        break; // Only events raised by the application...

    }

    if (type == EVENT_TYPE_BUTTON_PRESS)

      switch (ident)

      {

        case bt_start:  // if start button was pressed...

            /* ... */

      }

  }

}
