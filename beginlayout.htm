beginlayout

    

RTC Function

Starts the definition of a layout.

Declaration:

void beginlayout (int type, [int margin], [int space], [bool scrolling])

Group:

Touch screen programming

Parameters:

type
margin
space
scrolling

Type of the layout
Optional width of border
Optional spacing of the contained objects
Optional activation of a scrolling area

Return value:

-

 

The instruction beginlayout starts the definition of a layout as a group of associated objects, i.e. all following objects are treated as a logical entity until the group is closed by the instruction endlayout. The parameter type specifies the layout type as follows:

Type

Alias

Description

1

LAYOUT_TYPE_VBOX

The objects in the layout are aligned vertically,

2

LAYOUT_TYPE_HBOX

The objects in the layout are aligned horizontally.

3

LAYOUT_TYPE_GRID

The objects in the layout are arranged in a grid. The individual grid rows are formed using the instructions beginrow and endrow.

The optional parameters margin and space define a gap in pixels. margin defines an outer frame around the layout as a gap to the neighbor objects in the superordinate layout, whereas space defines the gap between the individual objects within the layout. If -1 is passed as value for margin or space, the default values are used instead.

If the optional parameter scrolling is set to true (default = false), a horizontal and/or vertical scrolling area is automatically created as soon as the size of the objects within the layout exceeds the available space.

 

Example:

enum

{

  PAGE_1,

  PAGE_2,

}

 

enum

{

  ID_BUTTON_0,

  ID_BUTTON_1,

  ID_BUTTON_2,

  ID_BUTTON_3,

  ID_BUTTON_4,

  ID_BUTTON_5,

}

 

void CreatePage (void) 

{

  begindraw();

  createpage (PAGE_1);

  page_background (PAGE_1, "#d0e0ff");  // SysTec Default

  // Create outer vertical layout

  beginlayout (LAYOUT_TYPE_VBOX);

    // First line: three buttons horizontally aligned

    beginlayout (LAYOUT_TYPE_HBOX);

      createwidget (ID_BUTTON_0, WIDGET_TYPE_BUTTON);

      createwidget (ID_BUTTON_1, WIDGET_TYPE_BUTTON);

      createwidget (ID_BUTTON_2, WIDGET_TYPE_BUTTON);

    endlayout();  

    // Second line: two buttons horizontally aligned

    beginlayout (LAYOUT_TYPE_HBOX);

      createwidget (ID_BUTTON_3, WIDGET_TYPE_BUTTON);

      createwidget (ID_BUTTON_4, WIDGET_TYPE_BUTTON);

    endlayout();  

  endlayout(); // close outer layout  

  enddraw();

}

 

The five objects are arranged in the layout as follows:

Button 0

Button 1

Button 2

Button 3

Button 4

Notes:

Layouts may be nested, i.e. a layout may again contain layouts (see example).

Applicable preprocessor constants (e.g. LAYOUT_TYPE_VBOX) are available in the file "qtmonitor.h."
