jsonfrommsg

    

RTC Function

Transfers a statement from the message queue to a JSON statement list.

Declaration:

bool jsonfrommsg (int id, string *data)

Group:

JSON interface

Parameters:

id
data

Logical reference number of the JSON statement list
Variable for transferring metadata

Return value:

Function result

 

The function jsonfrommsg reads a JSON statement from the message queue. The availability of a not yet handled message matching the filter settings (set-up by the instruction msgsetfilter) can be checked beforehand using the function msgreceived. If the instruction can be read successfully, it is written into the JSON statement list specified by the parameter id for subsequent evaluation. Possibly available additional data, e.g. information about how to interpret the statement, is copied into the provided string variable data. If the function can be executed successfully, jsonfrommsg returns true as function result; otherwise the result is false.

 

Example:

xmlfrommsg (JSON_HANDLE, data);

jsongetvalue (JSON_HANDLE, "cmd", cmd);

if(cmd == "current-state")

{

  // send all data again...

}

 

Note:

The statement list is automatically cleared before new content is written to it. Calling jsonclear is not necessary.
