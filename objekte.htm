The Programming Language RTC     Touch Screen Programming

Objects

    

An object (widget) is created using createwidget. When created, the object is automatically assigned to the currently selected page or window. With the creation also the type of the object is specified, e.g. if it is a button, a table or whatsoever. If the object is a list or table object, the number of entries or the number of table rows and columns need to be specified using the instruction widget_attribute.

Where applicable, further properties of the object can be specified. Colors can be set using widget_background and widget_foreground. Possible text can be formatted using widget_fontface and widget_fontsize. If the object is a text input/output field, a list or a table, the operator prompt can be set with widget_text and aligned with widget_alignment. If it is a text input, the input format (e.g. numbers only) can be specified using widget_format.

Also the size of the object can be specified. With widget_dimensions the minimum width and height of the object is set whereas widget_size_policy is used to determine whether the object size can be changed. The arrangement of the object among its neighbors can be specified using widget_content_margins.

If an object should not be available at any time but only according to certain conditions (e.g. a start button), the function widget_enable can be used to activate/deactivate the object. Deactivated objects are shown in gray and cannot be activated. Furthermore, if the object should become invisible instead being shown in gray the instructions widget_hide and widget_show can be used.
