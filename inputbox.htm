Input dialog for variables.

The instruction inputbox generates a complete input dialog according to the example below. As parameter the instruction is given a text for the title bar after which – enclosed by a pair of curved brackets – the variable inputs are specified. Each input consists at least of a prompt and the variable name var. If applicable, the variable may also be declared within the function by putting the declaration word in front of it.

The function inputbox displays text in black proportional font at a size of 13 pixels. Variables are shown in a monospaced font, bold. During entry, the background of a variable changes to blue.

The entry of the individual variables fully complies with the function input, including the option to use the F5 key to call up additional help information that previously has been assigned to the variable via the instruction setinfo.

If the operator prompt prompt is appended a list of options, enclosed in a pair of curved brackets, the input field complies with the function inputselection, i.e. the operator can choose an option from the provided list using the horizontal cursor keys. The number of the chosen option within the list is written as result into the target variable.

As an alternative to normal variables, the inputbox may also contain index fields of database tables. In that case, the entry fully complies with the function select, including the option to press the F3 key to choose a record from a list.

The inputbox may precede an optional if statement with expr being a Boolean expression of any complexity. If the expression returns false, the input is skipped. For visualization, the prompts of disabled inputs are written with light gray text.

Furthermore, the variable may be appended an optional until statement. Again, expr is a Boolean expression of any complexity that determines whether the entered data is accepted.

If applicable, the various function keys (soft keys) may be assigned a function. To do so, the number of the function key is entered, followed by the key label and the name of a function without parameters that is to be executed.

 

Example:

void GeneralSettings (void)

{

  inputbox ("Settings\\General")

  {

   "Date"           date;

   "Time"           time;

   "Ticket No."     TicketNo;

   "Cons. No."      ConsNo;

   "Min. load"      MinLoad until (MinLoad > 0);

   if (Printer)

     "Printer{off,COM1,COM2,USB}" PrinterPort;

   "With host"      WithHost;

   "Password"       UserPassword;

 

   fct2 "Print"     PrintSetting();

  }

}

 

 

Notes:

The instruction inputbox provides multi-language operator prompting; the use of the function lsub is not necessary.

If the dialog contains more than 7 variable inputs, the dialog is arranged on multiple pages. Triangles on the left indicate preceding or subsequent pages.

Variables declared inside the inputbox statement are stored in the nov RAM.

The instruction inputbox does not save the original content of the application screen or the previously selected font attributes. This has to be considered in the design of the program.
