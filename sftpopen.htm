sftpopen

    

RTC Function

Opens a connection to an SFTP server.

Declaration:

bool sftpopen (int sfno, string ipaddr, int port, string user, string pw)

Group:

File handling

Parameters:

sfno
ipaddr
port
user
pw

Logical connection number
IP address of SFTP server
SFTP port
User name
User password

Return value:

Function result

 

The function sftpopen opens a secured connection to an SFTP server and makes it available for subsequent file access (e.g. via sftpget) under the logical connection number specified by the parameter fno. The logical connection number sno can freely be chosen from the range 0 to 7.

The parameter ipaddr contains the IP address of the SFTP server as string, whereas the integer parameter port defines the port of the SFTP server. The two parameters user and pw specify the user name and password to log-in on the given server. If the connection can be opened successfully, sftpopen returns true as function result; otherwise the result is false. The connection status can be checked using the instruction sftpconnected.

After file access is completed, the connection to the SFTP server should be closed using sftpclose.

 

Example:

/* Get order file from FTP server */

 

#define SFTP_PORT 22

 

bool GetOrders (void)

{

bool result = false;

 

  if (sftpopen (0, ServerIP, SFTP_PORT))  // Open FTP connection...

  {

    // get new orders and copy to 'shared'...

    if (ftpget (0, "Production/Scale_1/Orders.csv", "shared/Orders.csv"))  

    {

      result = true;

      // delete order file from server

      if (! ftpremove (0, "Production/Scale_1/Orders.csv"))

        ShowErrorMsg ("SFTP access denied!");

    }

    else

      ShowErrorMsg ("No new orders available!");

  }

  else

    ShowErrorMsg ("SFTP access denied!");

 

  if (!ftpclose (0)) // Close FTP connection...

    ShowErrorMsg ("SFTP access denied!");      

  }

  else

    ShowErrorMsg ("SFTP server not available!");

  return (result);

}

 

Notes:

Depending on type and load of the network, opening an SFTP connection may take several seconds. The programmer should respect this in his program design.

If the SFTP connection is already open when sftpopen is executed, it is automatically closed and reopened anew.
