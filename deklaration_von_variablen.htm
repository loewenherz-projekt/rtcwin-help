The Programming Language RTC     Variables and Constants

Declaration of Variables

    

Variables are reserved memory locations with variable content. In a program, variables are referred to by their names. Prior to the first usage, a variable has to be declared:

Type  Name

Type is the data type specifying the value facet of the variable; Name is the name of the variable. Valid variable declarations are:

int i;

Declaration of an integer variable with name i

float pi;

Floating-point variable with name pi

double CorrFactor

Floating-point variable with double precision with name CorrFactor

decimal Total%8.2;

Formatted floating-point variable Total with 2 trailing decimals and an overall length of 8 characters

bool WithPrinter;

Boolean variable

string RxBuffer[80];

String variable RxBuffer with a maximum length of 80 characters

char ch;

Variable ch for a single character

timer t1;

Variable t1 for timer functions

A format specification for data type decimal as well as length specification for data type string is mandatory. For the data types int, float and double, format specifications are optional.

The thus declared variables occupy memory space in RAM and need to be initialized after power-up or before first usage. However, by putting the statement nov in front of the declaration a variable can be designed as non-volatile. Non-volatile variables are located in battery-backed memory and do not loose contents at power-down.

nov   Type  Name

 

Where variable contents must not be lost even in case of a failure of the battery backup, these variables may be stored in the flash EPROM by putting the statement flash in front of the declaration.

flash Type Name

The thus declared variables have to be copied to the battery-backed memory using the instruction flashload before they can be used by subsequent operations. The instruction flashsave has to be used to store the variables again in the flash memory, so they will be saved permanently and independently of the backup battery.

When using multitasking, variables might be needed independently of functions but dependent from the individual tasks. Such variables can be declared task local by putting the statement loc in front of the type statement. The thus declared variables may be used in various tasks, however assign different physical memory areas for each task. Thus, they cannot be accessed from another task.

loc   Type  Name

 

With one declaration word, multiple variables of the same type may be declared in one stroke, e.g.:

 int i, j;

Declaration of two integer variables with name i and j

 

Also, an initial value can be assigned to a variable during declaration:

 int i = 0;

Declaration of an integer variable with name i and initial value 0.

 float pi = 3.1416;

Declaration of the variable pi including initialization

Local variables declared within a function – they are only valid within this function – can simply be declared during first use:

void InputOrderNo (void)

{

  showtext (10, 22, "Order No.");

  char ch = input (100, 22, OrderNo);

}
