The Programming Language RTC     Multitasking

General

    

The RTC operating system gives the programmer the option to run several tasks in parallel as it where at the same time. For this purpose, two different algorithms are available:

·In parallel to the main application, a second thread is run every 10 milliseconds that accomplishes the update of internal system variables. This thread can be assigned further functions by the application program. The benefit of having one fix background task consists in the fix time frame that guarantees a predictable reaction time and eliminates drawbacks on the real-time behavior of the system.
·Alternatively, any own function may be started as a task and run in parallel by the operating system. This allows the programmer to design this task independent from the application. However, a predictable reaction time on any event and thus a reliable real time behavior cannot be guaranteed anymore. Rather, system reaction time depends on the number and load of the individual tasks.

Generally, both types of multitasking require that simultaneous access to system resources has to be avoided or, as a minimum, should be synchronized through semaphore variables. Especially reading the keyboard or output to the display should be accomplished by one task only, typically the main task.
